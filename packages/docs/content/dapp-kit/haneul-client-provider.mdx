---
title: HaneulClientProvider
---

The `HaneulClientProvider` manages the active `HaneulClient` that hooks and components use in the dApp
Kit.

## Usage

Place the `HaneulClientProvider` at the root of your app and wrap all components that use the dApp Kit
hooks.

`HaneulClientProvider` accepts a list of network configurations to create `HaneulClient` instances for the
currently active network.

```tsx
import { createNetworkConfig, HaneulClientProvider, WalletProvider } from '@haneullabs/dapp-kit';
import { getFullnodeUrl } from '@haneullabs/sui/client';

// Config options for the networks you want to connect to
const { networkConfig } = createNetworkConfig({
	localnet: { url: getFullnodeUrl('localnet') },
	mainnet: { url: getFullnodeUrl('mainnet') },
});

function App() {
	return (
		<HaneulClientProvider networks={networkConfig} defaultNetwork="localnet">
			<YourApp />
		</HaneulClientProvider>
	);
}
```

## Props

- `networks`: A map of networks you can use. The keys are the network names, and the values can be
  either a configuration object (`HaneulClientOptions`) or a `HaneulClient` instance.
- `defaultNetwork`: The name of the network to use by default when using the `HaneulClientProvider` as
  an uncontrolled component.
- `network`: The name of the network to use when using the `HaneulClientProvider` as a controlled
  component.
- `onNetworkChange`: A callback when the active network changes.
- `createClient`: A callback when a new `HaneulClient` is created (for example, when the active network
  changes). It receives the network name and configuration object as arguments, returning a
  `HaneulClient` instance.

## Controlled component

The following example demonstrates a `HaneulClientProvider` used as a controlled component.

```tsx
import { createNetworkConfig, HaneulClientProvider } from '@haneullabs/dapp-kit';
import { getFullnodeUrl } from '@haneullabs/sui/client';
import { useState } from 'react';

// Config options for the networks you want to connect to
const { networkConfig } = createNetworkConfig({
	localnet: { url: getFullnodeUrl('localnet') },
	mainnet: { url: getFullnodeUrl('mainnet') },
});

function App() {
	const [activeNetwork, setActiveNetwork] = useState('localnet' as keyof typeof networks);

	return (
		<HaneulClientProvider
			networks={networkConfig}
			network={activeNetwork}
			onNetworkChange={(network) => {
				setActiveNetwork(network);
			}}
		>
			<YourApp />
		</HaneulClientProvider>
	);
}
```

## HaneulClient customization

The following example demonstrates how to create a custom `HaneulClient`.

```tsx
import { HaneulClientProvider } from '@haneullabs/dapp-kit';
import { getFullnodeUrl, HaneulClient, HaneulHTTPTransport } from '@haneullabs/sui/client';

// Config options for the networks you want to connect to
const networks = {
	localnet: { url: getFullnodeUrl('localnet') },
	mainnet: { url: getFullnodeUrl('mainnet') },
} satisfies Record<string, HaneulClientOptions>;

function App() {
	return (
		<HaneulClientProvider
			networks={networks}
			defaultNetwork="localnet"
			createClient={(network, config) => {
				return new HaneulClient({
					transport: new HaneulHTTPTransport({
						url: 'https://api.safecoin.org',
						rpc: {
							headers: {
								Authorization: 'xyz',
							},
						},
					}),
				});
			}}
		>
			<YourApp />
		</HaneulClientProvider>
	);
}
```

## Using the HaneulClient from the provider

To use the `HaneulClient` from the provider, import the `useHaneulClient` function from the
`@haneullabs/dapp-kit` module.

```tsx
import { useHaneulClient } from '@haneullabs/dapp-kit';

function MyComponent() {
	const client = useHaneulClient();

	// use the client
}
```

## Creating a network selector

The dApp Kit doesn't provide its own network switcher, but you can use the `useHaneulClientContext`
hook to get the list of networks and set the active one:

```tsx
function NetworkSelector() {
	const ctx = useHaneulClientContext();

	return (
		<div>
			{Object.keys(ctx.networks).map((network) => (
				<button key={network} onClick={() => ctx.selectNetwork(network)}>
					{`select ${network}`}
				</button>
			))}
		</div>
	);
}
```

## Using network specific configuration

If your dApp runs on multiple networks, the IDs for packages and other configurations might change,
depending on which network you're using. You can use `createNetworkConfig` to create per-network
configurations that your components can access.

The `createNetworkConfig` function returns the provided configuration, along with hooks you can use
to get the variables defined in your configuration.

- `useNetworkConfig` returns the full network configuration object
- `useNetworkVariables` returns the full variables object from the network configuration
- `useNetworkVariable` returns a specific variable from the network configuration

```tsx
import { createNetworkConfig, HaneulClientProvider, WalletProvider } from '@haneullabs/dapp-kit';
import { getFullnodeUrl } from '@haneullabs/sui/client';
import { QueryClient, QueryClientProvider } from '@tanstack/react-query';

// Config options for the networks you want to connect to
const { networkConfig, useNetworkVariable } = createNetworkConfig({
	localnet: {
		url: getFullnodeUrl('localnet'),
		variables: {
			myMovePackageId: '0x123',
		},
	},
	mainnet: {
		url: getFullnodeUrl('mainnet'),
		variables: {
			myMovePackageId: '0x456',
		},
	},
});

const queryClient = new QueryClient();

function App() {
	return (
		<QueryClientProvider client={queryClient}>
			<HaneulClientProvider networks={networkConfig} defaultNetwork="localnet">
				<WalletProvider>
					<YourApp />
				</WalletProvider>
			</HaneulClientProvider>
		</QueryClientProvider>
	);
}

function YourApp() {
	const id = useNetworkVariable('myMovePackageId');

	return <div>Package ID: {id}</div>;
}
```
